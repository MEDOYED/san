                        FUNKCJE DOSTĘPNOŚCI (WCAG 2.1)
═══════════════════════════════════════════════════════════════════════════════

✓ Semantyczny HTML5
  • Użycie odpowiednich znaczników: <header>, <nav>, <main>, <article>, <footer>
  • Role ARIA: banner, navigation, main, contentinfo

✓ Nawigacja klawiaturowa
  • Pełna obsługa klawiatury (Tab, Enter, Escape)
  • Logiczna kolejność tabulacji
  • Widoczne wskaźniki fokusa

✓ Skip link
  • "Przejdź do głównej treści" - ukryty link dla czytników ekranu
  • Dostępny po naciśnięciu Tab

✓ ARIA atrybuty
  • aria-label - etykiety dla elementów nawigacji
  • aria-current="page" - zaznaczenie aktualnej strony
  • aria-labelledby - powiązanie nagłówków z sekcjami
  • aria-expanded - stan rozwiniętego menu

✓ Alt teksty dla obrazów
  • Wszystkie obrazy mają opisowe atrybuty alt
  • Figcaptions zawierają dodatkowy kontekst

✓ Kontrast kolorów
  • Współczynnik kontrastu min. 4.5:1 (WCAG AA)
  • Testowane dla normalnego i dużego tekstu
  • Wsparcie dla trybu wysokiego kontrastu

✓ Responsywność
  • Mobile-first design
  • Breakpoint: 768px
  • Responsywne menu mobilne z hamburgerem

✓ Fokus wizualny
  • Outline: 3px solid z wyraźnym kolorem (#ff6b35)
  • Offset: 2px dla lepszej widoczności
  • Różne kolory dla różnych stanów

✓ Interaktywne elementy
  • Wystarczająco duże obszary klikalne (min. 44x44px)
  • Hover i focus states dla wszystkich interaktywnych elementów
  • Smooth scroll dla lepszego UX

✓ Wsparcie dla preferencji użytkownika
  • prefers-reduced-motion - wyłączenie animacji
  • prefers-contrast: high - wysoki kontrast
  • Lazy loading obrazów

✓ Struktura nagłówków
  • Logiczna hierarchia (h1 → h2 → h3)
  • Jeden h1 na stronę
  • Nie pomijanie poziomów

✓ Formularze i linki
  • Opisowe teksty linków (nie "kliknij tutaj")
  • target="_blank" z rel="noopener noreferrer"
  • Visible focus states

✓ JavaScript
  • Progressive enhancement
  • Witryna działa bez JavaScript
  • Graceful degradation
